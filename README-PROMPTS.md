## 功能详解

### 📝 提示词管理

**完整的提示词创建、编辑、查看、删除功能**

#### 创建提示词

**操作步骤:**
1. 点击右上角 **"➕ 新建提示词"** 按钮
2. 弹出编辑表单,填写:
   - **标题**(必填):简洁明了的名称
   - **描述**(可选):说明提示词的用途
   - **内容**(必填):完整的提示词文本
   - **文件夹**(可选):选择分类文件夹
   - **标签**(可选):输入标签名,回车添加
3. 点击 **"保存"**
4. Toast 提示"创建成功",自动刷新列表

**提示:**
- 首次使用时会自动创建 3 个示例文件夹、3 个示例标签、3 个示例提示词,帮助您快速熟悉功能

#### 查看提示词详情

**两种方式:**
- **点击卡片**:打开详情模态框,查看完整内容
- **查看按钮**(眼睛图标):同样打开详情

**详情页包含:**
- 📋 元数据:文件夹、标签、更新时间
- 📝 描述:完整描述内容
- 📄 内容:保留格式的提示词全文(可复制)
- 🔄 版本历史:可展开查看和还原版本

#### 编辑提示词

**三种方式:**
1. **卡片编辑按钮**:鼠标悬停卡片,点击编辑图标
2. **详情页编辑按钮**:打开详情后点击底部"编辑"按钮
3. **URL 参数**:访问 `promptpro.html?edit=[promptId]` 直接打开编辑

**编辑时:**
- 表单预填当前数据
- 修改内容后保存
- **自动触发版本保存**(如果内容有变化)
- 版本号修订号自动递增(1.0.0 → 1.0.1)

#### 删除提示词

**两种方式:**
1. **卡片删除按钮**:鼠标悬停卡片,点击删除图标
2. **详情页删除按钮**:打开详情后点击底部"删除"按钮

**删除操作:**
- 弹出确认对话框
- 确认后执行**级联删除**:
  - 删除提示词主记录
  - 删除所有标签关联
  - 删除所有历史版本
- Toast 提示"已删除"

#### 收藏提示词

**操作方式:**
- **卡片收藏图标**:点击切换收藏状态
- **导航栏视图**:点击"收藏"按钮,只显示已收藏的提示词

### 🔄 版本控制与对比

**完整的版本管理:自动保存、差异对比、一键还原**

#### 自动版本保存

**触发时机:**
- 编辑提示词时,**内容发生变化**自动触发
- 只修改标题、描述、文件夹、标签**不会**触发版本保存

**版本规则:**
- 保存的是**修改前的旧内容**
- 语义化版本号自动递增(修订号 +1)
  - 示例:`1.0.0` → `1.0.1` → `1.0.2`
- 去重机制:同一秒内不重复保存相同内容
- `version_count` 自动 +1

**使用场景:**
```
编辑提示词 → 修改了内容 → 点击保存
        ↓
系统自动保存旧版本到版本历史
        ↓
创建新版本 v1.0.1,保留当前内容
        ↓
您可以随时还原到任意版本
```

#### 版本历史查看

**查看方式:**
1. 点击提示词卡片打开详情
2. 滚动到"版本历史"区域
3. 点击折叠箭头展开版本列表

**版本列表显示:**
- 按时间**倒序**排列(最新在前)
- 每项显示:
  - 版本号(等宽字体,绿色标签)
  - 当前版本标记("当前"徽章)
  - 创建时间
  - 内容大小(KB)
- 操作按钮:
  - **查看**(眼睛图标):查看该版本完整内容
  - **还原**(历史图标):还原到该版本(当前版本不显示此按钮)

**版本历史 HTML 示例:**
```
🔄 版本历史 (3)              [版本差异对比] ▼
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
☑️ v1.2.0 当前    2026-04-09 10:30   2.5 KB   [👁️ 查看]
☐️ v1.1.0         2026-04-08 15:20   2.3 KB   [👁️ 查看] [⏪ 还原]
☐️ v1.0.0         2026-04-07 09:10   2.0 KB   [👁️ 查看] [⏪ 还原]
```

#### 版本差异对比(🔥 核心功能)

**一键对比两个版本,清晰展示修改内容**

**操作步骤:**
1. 展开版本历史列表
2. **勾选两个版本**的 checkbox(任意两个版本)
3. "版本差异对比"按钮变为可用(高亮显示)
4. 点击按钮,弹出对比模态框

**对比模态框内容:**
```
┌──────────────────────────────────────────────┐
│           版本差异对比                    [✕] │
├──────────────────────────────────────────────┤
│  较早版本         ➡️         较新版本         │
│  v1.0.0                    v1.2.0           │
│  2026-04-07 09:10          2026-04-09 10:30  │
├──────────────────────────────────────────────┤
│  +2 行新增   -1 行删除   15 行未变更          │
├──────────────────────────────────────────────┤
│   1  |  你是一个AI助手                       │
│   2  |  请用中文回答                         │
│ + 3  |  回答要简洁明了                       │
│ - 4  |  回答要详细                           │
│   5  |  必要时提供代码示例                    │
│  ... |  ...                                  │
└──────────────────────────────────────────────┘
```

**差异高亮显示:**
- 🟢 **绿色背景** = 新增行(`+ ` 前缀)
- 🔴 **红色背景** = 删除行(`- ` 前缀)
- ⚪ **普通背景** = 未变更行(`  ` 前缀)
- 每行显示行号和内容

**关闭方式:**
- 点击右上角关闭按钮
- 或点击模态框外部遮罩层

**使用场景:**
```
编辑提示词多次后 → 想查看改了什麽
        ↓
勾选 v1.0.0 和 v1.2.0
        ↓
点击"版本差异对比"
        ↓
清晰看到新增了"回答要简洁明了"
            删除了"回答要详细"
```

#### 版本还原

**一键还原到任意历史版本,当前版本自动保留**

**操作步骤:**
1. 展开版本历史列表
2. 点击目标版本的 **"⏪ 还原"** 按钮
3. 弹出确认对话框:
   ```
   确认还原到版本 v1.0.0?
   
   当前内容将作为新版本保存,
   提示词内容将替换为目标版本的内容。
   
   [取消]  [确认还原]
   ```
4. 点击"确认还原"
5. 系统自动:
   - 保存当前内容为新版本
   - 将内容替换为目标版本内容
   - 更新 `current_version` 为目标版本号
6. Toast 提示"已成功还原到版本 X.X.X"

**还原特性:**
- ✅ **安全还原**:当前内容自动保存为新版本,不会丢失
- ✅ **版本保留**:所有历史版本依然存在
- ✅ **版本号更新**:还原后 `current_version` 更新为目标版本号

**使用场景:**
```
编辑提示词 → 改得不满意 → 想回到之前的版本
        ↓
找到满意的版本 v1.0.0 → 点击"还原"
        ↓
确认还原 → 当前内容保存为 v1.3.0
        ↓
提示词内容恢复为 v1.0.0 的内容
        ↓
current_version 更新为 v1.0.0
```

### 📁 文件夹分类

**层级化管理提示词,灵活组织您的提示词库**

#### 创建文件夹

**操作步骤:**
1. 点击左侧边栏底部 **"➕ 新建文件夹"** 按钮
2. 弹出编辑框,输入文件夹名称
3. 点击"保存"或按回车
4. Toast 提示"创建成功"

#### 文件夹折叠/展开

**操作方式:**
- 点击"全部"行右侧的 **箭头图标(▶)**
- 展开:显示所有文件夹列表
- 折叠:只显示"全部"选项

**使用场景:**
- 文件夹很多时,折叠起来节省空间
- 需要切换文件夹时,展开快速选择

#### 切换文件夹过滤

**操作方式:**
- 点击左侧边栏任意文件夹
- 主内容区只显示该文件夹下的提示词
- 点击"全部",显示所有提示词

#### 删除文件夹

**操作方式:**
1. 点击文件夹右侧的 **删除图标(×)**
2. 弹出确认对话框
3. 点击"确认删除"

**删除特性:**
- ✅ **不级联删除**:文件夹删除**不会**影响提示词
- ✅ **提示词保留**:该文件夹下的提示词变为"未分类"状态(`folder_id = null`)
- ✅ **安全删除**:只有文件夹记录被删除

#### 使用场景

**场景 1:按用途分类**
```
创建文件夹:
├ AI工具 - 各种AI工具的提示词
├ 翻译 - 多语言翻译提示词
├ 编程 - 代码生成、调试提示词
└ 写作 - 文章写作、文案提示词
```

**场景 2:按项目分类**
```
创建文件夹:
├ 项目A - 项目A相关的所有提示词
├ 项目B - 项目B相关的所有提示词
└ 个人 - 个人使用的提示词
```

### 🏷️ 标签系统

**灵活标记提示词,多选过滤快速定位**

#### 创建标签

**操作步骤:**
1. 点击左侧边栏标签板块 **"➕ 新建标签"** 按钮
2. 弹出编辑框,输入标签名称
3. 点击"保存"或按回车
4. Toast 提示"创建成功"

#### 标签多选过滤

**操作方式:**
1. 左侧边栏显示所有**有使用次数**的标签(3列网格布局)
2. **点击标签**进行多选/取消选择
3. 多个标签之间是 **AND 关系**:提示词必须包含**所有**选中的标签
4. 选中后自动过滤主内容区

**过滤逻辑:**
```
选中标签: "AI" + "翻译"
        ↓
只显示同时包含这两个标签的提示词
        ↓
提示词必须既有"AI"标签,又有"翻译"标签
```

**清除过滤:**
- 再次点击已选中的标签,取消选择
- 或点击筛选栏的"清除所有"按钮

#### 编辑时标签管理

**在创建/编辑提示词时:**

**标签输入框:**
- 显示已选标签为**绿色芯片**,可点击 × 删除
- 输入标签名后**按回车**添加
- 输入时自动匹配已有标签,显示**前 5 个匹配项**(自动补全)
- 输入不存在的标签名,保存时**自动创建**

**标签建议:**
```
输入: "py"
        ↓
显示建议:
┌──────────────┐
│ Python       │
│ PyTorch      │
│ Pygame       │
└──────────────┘
```

#### 删除标签

**操作方式:**
1. 点击标签芯片上的 **删除图标(×)**
2. 弹出确认对话框
3. 点击"确认删除"
4. 删除标签及其**所有关联关系**
5. Toast 提示"已删除"

**删除影响:**
- 标签被删除
- 所有提示词的该标签关联被删除
- 提示词本身**不会被删除**

#### 使用场景

**场景 1:按技术栈标记**
```
标签: AI, GPT-4, Python, 翻译, 代码生成

提示词"翻译Python代码为GPT-4提示":
标签 = AI + GPT-4 + Python + 翻译

搜索时选中"AI" + "Python":
只显示同时包含这两个标签的提示词
```

**场景 2:按语言标记**
```
标签: 中文, 英文, 日文

需要找中英翻译提示词:
选中"中文" + "英文"
只显示同时包含这两个标签的提示词
```

### 🔍 智能搜索

**权重评分算法,精准匹配您需要的提示词**

#### 关键词搜索

**操作方式:**
1. 在顶部导航栏搜索框输入关键词
2. **防抖 500ms** 后自动搜索(输入完等待半秒)
3. 结果按**相关性分数降序**排列

**搜索范围:**
- ✅ 标题
- ✅ 描述
- ✅ 内容
- ✅ 标签
- ✅ 文件夹名称

#### 权重评分算法

**评分权重(从高到低):**
```
标题完全匹配:      10000 分  (最佳匹配)
标题前缀匹配:       8000 分
标题包含匹配:       5000 分
标签包含匹配:       2000 分
描述前缀匹配:       1500 分
描述包含匹配:       1000 分
文件夹包含匹配:      800 分
内容包含匹配:        300 分
```

**额外奖励:**
```
标题包含所有关键词: 每个关键词 +5000 分
标签包含所有关键词: 每个关键词 +3000 分
描述包含所有关键词: 每个关键词 +1500 分
跨字段匹配(2+字段): 每个字段 +1000 分
```

**排序规则:**
- 有搜索关键词时:按相关性分数降序
- 无搜索关键词时:按更新时间降序(最新在前)

**搜索示例:**
```
搜索: "GPT-4 翻译"

分词: ["GPT-4", "翻译"]

提示词 A:
标题: "GPT-4 中英文翻译提示词"  ← 包含两个关键词
标签: GPT-4, 翻译              ← 包含两个关键词
分数: 10000 + 2000 + 5000 + 3000 = 20000 分

提示词 B:
标题: "AI 写作助手"
内容: "...使用 GPT-4 进行翻译..."  ← 内容中包含
分数: 300 + 300 = 600 分

结果: 提示词 A 排在前面(分数更高)
```

#### 组合过滤

**多种过滤条件组合使用:**

**过滤条件:**
- 📁 **文件夹过滤**:选择左侧文件夹,只显示该文件夹下的提示词
- 🏷️ **标签过滤**:选中标签,只显示包含所有选中标签的提示词(AND 关系)
- ⭐ **收藏视图**:点击导航栏"收藏",只显示已收藏的提示词
- 🔍 **关键词搜索**:输入关键词,按相关性排序

**筛选栏显示:**
当应用过滤条件后,筛选栏显示当前所有过滤条件:
```
筛选栏:
┌──────────────────────────────────────────┐
│ 🔍 关键词: GPT  📁 文件夹: AI工具        │
│ 🏷️ 标签: AI, 翻译         [✕ 清除所有]  │
└──────────────────────────────────────────┘
```

**清除过滤:**
- 点击"清除所有"按钮,一键重置所有过滤条件
- 或逐个清除:
  - 点击导航栏搜索框的 ×,清除关键词
  - 点击"全部"文件夹,重置文件夹过滤
  - 再次点击已选中的标签,取消标签过滤
  - 点击"全部提示词"按钮,重置收藏视图

#### 搜索 UI 交互

**防抖机制:**
- 输入关键词后,**等待 500ms** 才触发搜索
- 避免频繁搜索,提升性能
- 继续输入会重置计时器

**实时反馈:**
- 加载中:显示加载动画
- 无结果:显示空状态提示
- 有结果:立即渲染,按分数排序

### 💾 导入导出

**完整备份和恢复您的提示词数据**

#### 导出数据

**操作步骤:**
1. 点击左侧边栏数据管理板块 **"📤 导出数据"** 按钮
2. 系统自动生成 JSON 文件
3. 浏览器触发下载
4. 文件名格式:`promptpro-backup-YYYY-MM-DD.json`

**导出内容:**
导出的 JSON 文件包含**完整数据**:
```json
{
  "version": "2.0.0",
  "export_date": "2026-04-09T10:30:00.000Z",
  "prompts": [...],        // 所有提示词
  "folders": [...],        // 所有文件夹
  "tags": [...],           // 所有标签
  "tag_relations": [...],  // 所有标签关联
  "versions": [...]        // 所有版本历史
}
```

**使用场景:**
- 💾 **定期备份**:每周/每月导出一次,防止数据丢失
- 📦 **迁移数据**:换电脑或浏览器时,导出后导入
- 🔒 **数据存档**:重要提示词修改后导出存档

#### 导入数据

**操作步骤:**
1. 点击左侧边栏数据管理板块 **"📥 导入数据"** 按钮
2. 选择之前导出的 JSON 文件
3. 弹出确认对话框:
   ```
   确认导入数据?
   
   当前所有数据将被覆盖,
   此操作不可撤销。
   
   [取消]  [确认导入]
   ```
4. 点击"确认导入"
5. 系统:
   - 验证 JSON 文件格式
   - **清空所有存储**(覆盖模式)
   - 导入新数据到所有存储
   - 刷新整个界面
6. Toast 提示"导入成功"或"导入失败:错误信息"

**导入策略:**
- ⚠️ **覆盖模式**:先清空所有存储,再导入新数据
- 🔒 **数据验证**:检查 JSON 格式是否正确
- ❌ **错误处理**:JSON 解析失败或导入失败时,显示 Toast 错误提示,不修改现有数据

**使用场景:**
```
场景 1:恢复备份
之前导出了备份 → 数据丢失或损坏
        ↓
选择备份文件 → 确认导入
        ↓
数据恢复为备份时的状态

场景 2:迁移数据
旧电脑导出 JSON → 新电脑导入
        ↓
完整迁移所有提示词、文件夹、标签、版本历史
```

## 常见问题

### Q1: PromptPro 的数据存储在哪里?

**A:** 数据存储在浏览器的 **IndexedDB** 数据库中,数据库名为 `PromptProDB`。数据完全本地化,**不会上传到任何服务器**。

**查看方式:**
- 按 `F12` 打开开发者工具
- 切换到"应用"标签
- 展开"IndexedDB"
- 查看 `PromptProDB` 数据库

---

### Q2: 如何备份提示词?

**A:** 支持三种备份方式:
1. **本地文件夹备份**:在设置中授权本地目录,支持手动备份和每日自动备份
2. **百度网盘云端备份**:在设置中点击"连接百度网盘"授权后,支持自动/手动备份到云端
3. **手动导出**:点击左侧边栏 **"📤 导出数据"** 按钮,生成 JSON 文件下载

---

### Q3: 导入会覆盖现有数据吗?

**A:** 是的。导入采用**覆盖模式**:先清空所有存储,再导入新数据。导入前会弹出确认对话框,请谨慎操作。

---

### Q4: 版本历史会占用很多空间吗?

**A:** 版本历史只保存文本内容,每个版本的体积通常很小(几 KB)。如果版本过多,可以:
1. 删除不需要的提示词(会级联删除所有版本)
2. 或定期导出备份后重新导入(只保留需要的版本)

---

### Q5: 为什么编辑提示词后没有新版本?

**A:** 只有**内容**发生变化时才会触发版本保存。如果只修改标题、描述、文件夹或标签,**不会**创建新版本。

---

### Q6: 如何删除版本历史?

**A:** 当前不支持单独删除版本。如需清理版本历史,可以:
1. 还原到目标版本
2. 删除并重新创建提示词

---

### Q7: 标签可以多选吗?

**A:** 可以。左侧边栏的标签支持多选,多个标签之间是 **AND 关系**(提示词必须包含所有选中的标签)。

**示例:**
```
选中标签: "AI" + "Python"
        ↓
只显示同时包含这两个标签的提示词
```

---

### Q8: 搜索支持中文吗?

**A:** 支持。搜索算法按空格、中文标点、英文标点分隔关键词,完美支持中文搜索。

---

### Q9: 如何切换主题?

**A:** 点击导航栏的 **🌙/☀️ 图标**即可切换亮色/暗色主题。主题偏好保存到 `localStorage`,下次访问时自动加载。

---

### Q10: URL 参数如何使用?

**A:** 
- `promptpro.html?detail=[promptId]` - 打开提示词详情
- `promptpro.html?edit=[promptId]` - 打开编辑模态框

页面加载 500ms 后自动处理参数。

**使用场景:**
- 在其他地方保存提示词链接,点击直接打开详情或编辑
- 从书签或笔记中快速跳转到特定提示词

---

### Q11: 首次打开看到示例数据,如何删除?

**A:** 首次打开时自动创建 3 个示例文件夹、3 个示例标签、3 个示例提示词。您可以:
- **逐个删除**:点击删除按钮,确认删除
- **全部删除**:导入一个空的 JSON 文件(会覆盖所有数据)

---

### Q12: 支持云同步吗?

**A:** 支持百度网盘云端备份。在设置中点击"连接百度网盘"完成授权后,可将数据自动或手动备份到百度网盘 `/apps/FavsHub/` 目录。如需多设备同步,可以:
1. 设备 A 上传备份到百度网盘
2. 设备 B 从云端备份历史中下载备份文件
3. 设备 B 导入数据
